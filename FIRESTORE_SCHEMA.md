# Firestore Database Schema Guide

This document outlines the complete Firestore database structure for AlgoGuide Backend.

## Overview

Your Firestore database uses **3 main collections**:
1. `users` - Main user collection
2. `users/{user_id}/question_answers` - Subcollection for user answers
3. `home` - Collection for personalized learning resources

---

## Collection 1: `users`

**Path:** `users/{userId}`

**Purpose:** Stores basic user information

**Document Structure:**
```json
{
  "name": "John Doe",              // string (required)
  "email": "john@example.com",     // string (required, should be unique)
  "age": 25                         // number (optional)
}
```

**Example Document:**
```json
{
  "name": "John Doe",
  "email": "john@example.com",
  "age": 25
}
```

**Important Notes:**
- Document ID is auto-generated by Firestore
- Email is used for queries, so ensure uniqueness
- Age is optional and can be null

---

## Collection 2: `question_answers` (Subcollection)

**Path:** `users/{userId}/question_answers/{answerId}`

**Purpose:** Stores user's onboarding questionnaire answers

**Document Structure:**
```json
{
  "email": "john@example.com",                    // string (required)
  "answers": [                                    // array of objects (required)
    {
      "question_id": "primary_language",          // string
      "question_text": "What is your primary language?",  // string
      "answer": "Python"                          // string
    },
    {
      "question_id": "tech_stack",
      "question_text": "What technologies are you familiar with?",
      "answer": "React, Node.js, MongoDB"
    }
    // ... more answer objects
  ],
  "submitted_at": "2024-01-15T10:30:00Z"         // timestamp (auto-set)
}
```

**Answer Object Fields:**
- `question_id`: Unique identifier for the question (e.g., "name", "status", "education", "graduation_year", "primary_language", "tech_stack", "familiar_topics", "weak_areas", "target_companies", "preferred_role", "target_timeline", "preferred_resources")
- `question_text`: The actual question text
- `answer`: User's response to the question

**Example Document:**
```json
{
  "email": "john@example.com",
  "answers": [
    {
      "question_id": "name",
      "question_text": "What is your name?",
      "answer": "John Doe"
    },
    {
      "question_id": "primary_language",
      "question_text": "What is your primary programming language?",
      "answer": "Python"
    },
    {
      "question_id": "tech_stack",
      "question_text": "What technologies are you familiar with?",
      "answer": "React, Node.js, MongoDB"
    },
    {
      "question_id": "weak_areas",
      "question_text": "What are your weak areas?",
      "answer": "Dynamic Programming, Graph Algorithms"
    },
    {
      "question_id": "target_companies",
      "question_text": "Which companies are you targeting?",
      "answer": "Google, Microsoft, Amazon"
    },
    {
      "question_id": "preferred_role",
      "question_text": "What role are you targeting?",
      "answer": "Software Engineer"
    }
  ],
  "submitted_at": "2024-01-15T10:30:00Z"
}
```

**Important Notes:**
- Multiple answer submissions per user are allowed (creates multiple documents)
- Latest submission is used for resource generation
- `submitted_at` is automatically set by Firestore using `SERVER_TIMESTAMP`

## Collection 3: `home`

**Path:** `home/{homeId}`

**Purpose:** Stores personalized learning resources generated for users

**Document Structure:**
```json
{
  "user_id": "abc123xyz",                        // string (required) - references users/{userId}
  "user_profile": {                              // object (required)
    "name": "John Doe",
    "status": "Student",
    "education": "Computer Science",
    "graduation_year": "2024",
    "primary_language": "Python",
    "tech_stack": ["React", "Node.js", "MongoDB"],
    "familiar_topics": ["Arrays", "Strings"],
    "weak_areas": ["Dynamic Programming", "Graphs"],
    "target_companies": ["Google", "Microsoft"],
    "preferred_role": "Software Engineer",
    "target_timeline": "6 months",
    "preferred_resources": ["Video", "Blog"]
  },
  "search_queries": [                            // array of strings (required)
    "Dynamic Programming Python tutorial",
    "Google Software Engineer interview prep",
    "Graph algorithms practice problems"
  ],
  "total_resources": 45,                         // number (required)
  "resources": {                                 // object (required) - categorized resources
    "weak_areas_improvement": [                  // array of resource objects
      {
        "title": "Dynamic Programming Masterclass",
        "url": "https://youtube.com/watch?v=...",
        "description": "Complete guide to DP",
        "resource_type": "video",
        "difficulty": "intermediate",
        "estimated_time": 120,
        "tags": ["DP", "Algorithms"],
        "created_at": "2024-01-15T10:30:00Z"
      }
    ],
    "interview_preparation": [                   // array of resource objects
      {
        "title": "Google Interview Questions",
        "url": "https://leetcode.com/...",
        "description": "Practice Google interview questions",
        "resource_type": "practice",
        "difficulty": "advanced",
        "estimated_time": 60,
        "tags": ["Interview", "Google"],
        "created_at": "2024-01-15T10:30:00Z"
      }
    ],
    "skill_development": [],                     // array of resource objects
    "practice_problems": [],                     // array of resource objects
    "technology_tutorials": [],                  // array of resource objects
    "general_learning": []                       // array of resource objects
  },
  "generated_at": "2024-01-15T10:30:00Z",       // string (ISO format)
  "created_at": "2024-01-15T10:30:00Z"          // timestamp (auto-set)
}
```

**Resource Object Structure:**
```json
{
  "title": "Resource Title",                    // string
  "url": "https://example.com/resource",        // string
  "description": "Resource description",        // string
  "resource_type": "video|blog|course|documentation|practice|repository",  // string
  "difficulty": "beginner|intermediate|advanced",  // string
  "estimated_time": 30,                         // number (minutes)
  "tags": ["tag1", "tag2"],                     // array of strings
  "created_at": "2024-01-15T10:30:00Z",        // string (ISO format)
  "query": "original search query",            // string (optional)
  "source": "gemini_web_agent"                 // string (optional)
}
```

**Resource Categories:**
1. `weak_areas_improvement` - Resources to help with user's weak areas
2. `interview_preparation` - Interview prep materials, especially for target companies
3. `skill_development` - Resources for developing skills for preferred role
4. `practice_problems` - Coding problems, exercises, challenges
5. `technology_tutorials` - Tutorials for specific technologies
6. `general_learning` - Other valuable learning resources

**Important Notes:**
- Multiple home documents per user are allowed (creates new document each time resources are generated)
- Latest document is retrieved for user's home page
- `user_id` should reference an existing user in the `users` collection

## Required Firestore Indexes

To optimize queries, you may need to create composite indexes for:

1. **Query:** `users` collection where `email == {email}` (limit 1)
   - Index: Simple index on `email` field
   - Firestore will prompt you to create this automatically if needed

2. **Query:** `home` collection where `user_id == {user_id}`
   - Index: Simple index on `user_id` field
   - Firestore will prompt you to create this automatically if needed

**Note:** The code currently avoids needing indexes by:
- Fetching all documents and sorting in Python (not recommended for large datasets)
- Using simple equality queries that Firestore can handle efficiently

## How to Create Collections in Firestore

### Step 1: Create the `users` Collection

1. Open Firebase Console → Firestore Database
2. Click "Start collection" (if first time) or "Add collection"
3. Collection ID: `users`
4. Click "Next"
5. Add your first document:
   - Document ID: Click "Auto-ID" (or provide custom ID)
   - Add fields:
     - `name` (string): "John Doe"
     - `email` (string): "john@example.com"
     - `age` (number): 25 (optional)
6. Click "Save"

### Step 2: Create Subcollection (Automatically Created)

The `question_answers` subcollection will be created automatically when your backend code stores answers. To manually create it:

1. Click on a user document in `users` collection
2. Click "Start subcollection"
3. Collection ID: `question_answers`
4. Click "Next"
5. Add a sample document or wait for your backend to populate it

### Step 3: Create the `home` Collection

1. In Firestore Database, click "Add collection"
2. Collection ID: `home`
3. Click "Next"
4. Add your first document (or wait for backend to create it):
   - Document ID: Click "Auto-ID"
   - Add fields (see structure above)

## Security Rules (Recommended)

Add these security rules to your Firestore:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users collection
    match /users/{userId} {
      allow read: if request.auth != null || request.auth.uid == userId;
      allow write: if request.auth != null;
      
      // Question answers subcollection
      match /question_answers/{answerId} {
        allow read: if request.auth != null;
        allow write: if request.auth != null;
      }
    }
    
    // Home collection
    match /home/{homeId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && 
                     request.resource.data.user_id == request.auth.uid;
    }
  }
}
```

**Note:** Adjust these rules based on your authentication setup.

## API Endpoints That Use Each Collection

### `users` Collection:
- `GET /users` - Get all users
- `GET /users/{user_id}` - Get specific user
- `POST /users` - Create new user
- `PUT /users/{user_id}` - Update user
- `DELETE /users/{user_id}` - Delete user

### `question_answers` Subcollection:
- `POST /users/{email}/answers` - Store user answers
- `GET /users/{email}/answers` - Get user's answers

### `home` Collection:
- `POST /generate-resources/{user_id}` - Generate and store resources
- `POST /generate-resources-by-email/{email}` - Generate resources by email
- `GET /home/{user_id}` - Get user's home resources
- `GET /home-by-email/{email}` - Get home resources by email
